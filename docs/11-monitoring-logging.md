## 11.1 System Monitoring

### Health Check Endpoints

| Endpoint | Auth | Response | Purpose |
|----------|------|----------|---------|
| `GET /health` | None | `{ status: "ok" }` | Basic liveness probe |
| `GET /api/v1/status` | None | `{ database, redis, uptime }` | Infrastructure status |
| `GET /metrics` | API Key | Prometheus text format | Detailed performance metrics |

#### HTTP Metrics
| Metric | Type | Labels | Description |
|--------|------|--------|-------------|
| `http_request_duration_seconds` | Histogram | method, route, status | Request latency distribution |
| `http_requests_total` | Counter | method, route, status | Total request count |
| `http_request_size_bytes` | Histogram | method, route | Request body sizes |

## 11.2 Error Tracking

### Backend (Winston Logger)

#### Log Format
**Development:**
```
2026-03-11 14:30:45 [abc123-correlation-id] INFO: User logged in successfully
2026-03-11 14:30:46 [abc123-correlation-id] ERROR: Database query failed { stack: ... }
```

**Production (JSON):**
```json
{
  "timestamp": "2026-03-11T14:30:45.123Z",
  "level": "error",
  "correlationId": "abc123-def456",
  "message": "Database query failed",
  "stack": "Error: ...",
  "route": "/api/v1/reports",
  "method": "POST",
  "userId": "user-789"
}
```

#### Log Levels
| Level | Usage | Production Output |
|-------|-------|-------------------|
| `error` | Unhandled exceptions, critical failures | Always |
| `warn` | Degraded service, rate limits hit | Always |
| `info` | Key business events (login, report submission) | Always |
| `debug` | Detailed operation context | Dev only |

#### Correlation ID Tracing
- Generated by `tracer` middleware on every request
- Propagated via `X-Correlation-ID` header
- Injected into all Winston log entries
- Returned in response headers for client debugging
- Enables full request lifecycle tracing across services

### Mobile (Firebase Crashlytics)

#### Error Capture
```dart
// Global Flutter error handler
FlutterError.onError = (details) {
  FirebaseCrashlytics.instance.recordFlutterFatalError(details);
};

// Platform errors
PlatformDispatcher.instance.onError = (error, stack) {
  FirebaseCrashlytics.instance.recordError(error, stack, fatal: true);
  return true;
};
```

#### Tracked Events
| Event | Context |
|-------|---------|
| API call failures | Endpoint, status code, response body |
| Navigation errors | Route, stack trace |
| Service initialization failures | Service name, error details |
| Permission denials | Permission type, screen |
| Token refresh failures | Token state, HTTP status |

### Admin Dashboard (Console + Network)
- Browser console errors captured
- Axios interceptor logs failed API calls
- No dedicated error tracking service (see Missing Components)

## 11.3 Security Monitoring

### Authentication Monitoring
| Event | Log Level | Alert |
|-------|-----------|-------|
| Successful login | info | — |
| Failed login attempt | warn | After 5 failures from same IP |
| Token refresh | info | — |
| Token revocation | info | — |
| Account lockout (rate limit) | warn | Immediate |
| Admin login | info | Email notification |

### Anti-Replay Monitoring
| Event | Log Level | Alert |
|-------|-----------|-------|
| Replayed nonce detected | warn | After 3 attempts from same IP |
| Expired timestamp | warn | — |
| Redis unavailable (fail-closed) | error | Critical |

### Rate Limit Monitoring
| Limiter | Threshold for Alert |
|---------|-------------------|
| loginLimiter | >5 blocks/minute from same IP |
| reportLimiter | >3 blocks/hour from same user |
| featureLimiter | >10 blocks/hour from same user |

### Audit Log (Admin Actions)
All admin actions are logged immutably:
```json
{
  "id": "audit-001",
  "adminId": "admin-user-id",
  "action": "UPDATE_REPORT_STATUS",
  "targetType": "ScamReport",
  "targetId": "report-123",
  "payload": { "status": "VERIFIED", "previousStatus": "PENDING" },
  "createdAt": "2026-03-11T14:30:45.123Z"
}
```

### Content Moderation Monitoring
| Event | Log Level | Alert |
|-------|-----------|-------|
| PII detected in submission | warn | — |
| AI moderation flag | warn | — |
| Content flag submitted | info | Admin dashboard notification |
| 3+ flags on same content | warn | Admin review required |

## 11.4 Alerting Configuration

### Alert Channels
| Channel | Purpose | Configuration |
|---------|---------|---------------|
| Email | Critical system alerts | `ADMIN_ALERT_EMAIL` env var |
| Discord/Slack Webhook | Real-time alerting | `CRITICAL_ALERT_WEBHOOK_URL` env var |
| Admin Dashboard | Operational metrics | In-app stats |
| Firebase Crashlytics | Mobile crash reports | Firebase Console |

### Alert Severity Levels
| Severity | Response Time | Examples |
|----------|---------------|---------|
| **Critical** | Immediate | Database down, Redis down, API unreachable |
| **High** | <15 minutes | High error rate, auth brute force, disk >90% |
| **Medium** | <1 hour | Elevated latency, rate limit spikes, failed backups |
| **Low** | Next business day | Deprecated API usage, slow queries, minor warnings |
